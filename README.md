# 💻 LUCAS_OS v3.0

> Um portfólio interativo construído sob a arquitetura de um Sistema Operacional / Terminal retro. 

## 🧠 A Filosofia (The Core)

Este projeto não é apenas um site, é uma declaração de engenharia e gestão de produto. Como **Software Architect** e **Tech Lead**, construí este portfólio baseando-me em três pilares absolutos:

1. **Código não basta:** Se a arquitetura não resolve uma dor real de negócio e não tem viabilidade de escala, é apenas vaidade técnica.
2. **Dados são lei:** A interface pode mentir, o banco de dados não. Uma modelagem relacional forte previne o caos no front-end.
3. **Simplicidade Extrema no Front:** A complexidade e o caos devem ficar isolados nos bastidores (back-end/infra). A interface do usuário deve ser absurdamente simples e fluida.

## ⚙️ Arquitetura e Tech Stack

O `LUCAS_OS` foi construído com uma abordagem **Zero-Dependency** para o front-end. Nenhuma biblioteca inchada, nenhum framework reativo pesado. Apenas manipulação direta de DOM e CSS moderno para garantir performance máxima, acessibilidade e controle total da renderização.

* **HTML5:** Semântico e acessível.
* **CSS3:** Animações complexas (Keyframes), CSS Grid/Flexbox, Variáveis Nativas e transformações 3D.
* **JavaScript (Vanilla):** Lógica de jogos (Canvas API), física Drag & Drop, interpretador de comandos de terminal customizado e Intersection Observers.

## 🚀 Features de Destaque (Joyful UX)

O sistema foi desenhado para testar a curiosidade do usuário através de micro-interações e gamificação:

* **ZSH Terminal Engine:** Um terminal funcional integrado à interface. Digite `whoami`, `stack` ou procure pelos comandos secretos na base de código.
* **Física 3D CSS:** O crachá de identificação `[NÍVEL ROOT]` no Hero reage ao movimento do cursor com cálculos trigonométricos no JS e brilho holográfico.
* **Inventário de Skills (Drag & Drop):** Uma grade interativa usando a API nativa de Drag and Drop do HTML5, permitindo organizar as ferramentas de trabalho como módulos.
* **Engine de Mini-Games:** * **Snake Game:** Renderizado via `Canvas API` (2D Context), com física de colisão, crescimento de matriz e sistema de High Score salvo em memória.
  * **Memory Match:** Lógica de estados e grid interativo.
* **Boss Fight de Contato:** Uma mecânica de dano baseada em cliques para desbloquear a seção de contatos, com animações de UI Shake e Damage Numbers.
* **Tributo Oculto:** O Konami Code clássico (`U U D D L R L R B A`) foi implementado usando um algoritmo de *Sliding Window* à prova de falhas para desbloquear a lore do sistema.
